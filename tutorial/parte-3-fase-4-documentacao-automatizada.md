# Parte 3 — Fase 4: Documentação Automatizada

## Objetivo
Produzir documentação útil e sustentável a partir do código real.

## 1) Docstrings e comentários inteligentes
Use IA para:
- Documentar funções com regras não triviais
- Explicar decisões arquiteturais
- Evitar comentário redundante

## 2) README estruturado por contexto
No monorepo, separar documentação em:
- visão geral do sistema
- como rodar API e Web
- como configurar PostgreSQL
- como executar testes

## 3) Qualidade da documentação
Checklist:
- Reflete o estado atual do código
- Traz exemplos de uso reais
- Expõe limitações conhecidas
- Inclui troubleshooting básico

## Critérios de aceite
- Onboarding mais rápido
- Menos dúvidas repetidas no time
- Documentação sincronizada com evolução do código
